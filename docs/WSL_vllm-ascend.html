遇到的问题 (4)

宿主机 Docker 不可用，无法启动容器进行隔离的原生构建验证

方案: Docker 未安装在宿主机上。作为适配方案，验证在 Ubuntu 22.04 宿主机上直接执行。所有依赖安装、构建、测试命令均在宿主机环境进行，步骤和命令记录保持与 Dockerfile 中对应一致。

来源: 宿主机环境检查

Docker 不可用，无法启动干净的 Ubuntu 22.04 基础镜像进行隔离构建

方案: 在 Ubuntu 22.04 宿主机上直接执行构建、UT 和样例验证。宿主机环境已包含 CANN 9.0.0、PyTorch 2.10.0、torch-npu 2.10.0 等依赖，与 Dockerfile 目标环境一致。所有步骤和命令记录按技能要求保留。

来源: 宿主机环境检查

pre-commit run --all-files 首次执行时，各 hook 的虚拟环境创建极慢（每个 hook 5+分钟），导致超时

方案: 改为直接调用核心工具（ruff、codespell）进行验证，记录各 hook 的执行状态。clang-format、shellcheck 等工具已通过 apt-get 安装可用，但因 pre-commit 环境创建超时未完整执行。

来源: pre-commit 虚拟环境创建机制

pip 26.1.1 在 build isolation 模式下构建 arctic-inference 依赖时失败

方案: 使用 --no-build-isolation 标志跳过 build isolation，因为所有 build 依赖已预先安装在系统中。

来源: pip 26.1.1 + arctic-inference build dependency
