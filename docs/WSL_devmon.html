文档缺失 (2)

#1README 未提及 init.py 的 SDK 安装需要交互式确认许可协议（设置 OPENUBMC_LICENSE=true 可绕过），对非交互式环境不友好
#2README 仅指向外部 openUBMC 构建指导书链接，未在仓库内提供完整的依赖安装步骤

遇到的问题 (4)

init.py 的 SDK 安装步骤因非交互式运行导致 EOF 错误，无法自动完成

方案: 设置环境变量 OPENUBMC_LICENSE=true 并使用 echo Y 管道输入来绕过交互式确认

来源: bmcgo/utils/sdk_downloader.py 中的 input() 调用

bingo build 缺少 conan profile 文件（profile.luajit.ini、profile.dt.ini）

方案: 先从 default profile 创建基础版本，后通过完整 init.py 获取正式 profile

来源: bingo build 内部逻辑要求 profile.luajit.ini 存在

liblogger/[>=1.90.0]@openubmc/stable 包未在 openubmc_opensource 远程仓库中找到

方案: 完成完整的 init.py 安装流程后，liblogger 等依赖包自动从远程仓库下载并安装到本地缓存

来源: conanfile.py 通过 libmcpp 间接依赖 liblogger

2个性能基准测试（CsrPerfTest）失败：优化算法比正则算法慢

方案: 此为容器环境性能波动导致，非功能性缺陷。容器中 CPU 缓存、调度等因素可能导致微秒级性能比较不稳定

来源: libs/common/tests/test_csr_perf.cpp
