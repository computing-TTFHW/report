遇到的问题 (8)

Python 单元测试因缺少 NPU 硬件失败：ImportError: cannot import name 'npu_bridge_handle'

方案: 确认 Python 测试需要 NPU 硬件，在无 NPU 环境中跳过。这是预期行为。

来源: training/tf_rec_v1/python/tests/run_python_dt.sh

CMake TF_PATH 解析错误：python3.7 符号链接导致路径为 /usr 而非 /usr/local/python3.7.5

方案: 使用 readlink -f 解析真实路径

来源: test_ut.sh 中的 python_path 计算逻辑

链接错误：cannot find -lasan, cannot find -lascend_hal

方案: 安装 libasan (yum install libasan)，添加 devlib 目录到链接路径

来源: CMakeLists.txt 链接配置

CXX11 ABI 不兼容：googletest/emock 用新 ABI 编译，代码用旧 ABI

方案: 用 -D_GLIBCXX_USE_CXX11_ABI=0 重新编译 googletest 和 emock

来源: CMakeLists.txt add_compile_options

ASan (AddressSanitizer) 与 CANN 库不兼容导致 Illegal instruction

方案: 从 Debug 编译标志中移除 -fsanitize=address 和 -fsanitize-recover=address,all

来源: CMakeLists.txt CMAKE_CXX_FLAGS_DEBUG

CANN 库运行时符号缺失：libgraph_base.so 找不到 GetTypeId 等符号

方案: 使用 LD_PRELOAD 预加载 libge_common.so, libge_common_base.so, libgraph.so, libgraph_base.so

来源: CANN 9.0.0 运行时库依赖

部分 C++ 测试因 NPU 硬件依赖崩溃 (Illegal instruction / SIGSEGV)

方案: 使用 --gtest_filter 跳过需要 NPU 的测试 (CheckpointTest, EmbeddingDDRTest, HdTransferTest, KeyProcessTest 等)

来源: C++ 测试代码中的 NPU API 调用

CkptDataHandlerTest.KeyFreqMapCkpt 测试失败：对 moved-from 对象 tranData.datasetSize 的断言读取到未初始化内存

方案: 移除对 moved-from 对象的无效断言。SetDataset 内部执行 transferData = move(loadedData)，移动后原对象状态未定义。改用独立的 tranData2 对象进行第二次 SetDataset 调用。

来源: training/tf_rec_v1/src/tests/ckpt_data_handler/ckpt_data_handler_test.cpp
